LSL - Lightweight Subsystem for Linux

A CLI for managing lightweight Docker containers.

Usage | Configuration | License | GitHub

Overview

LSL aims to streamline the management of virtualized Linux-based development environments, making it easier and faster for software engineers, system administrators, and DevOps teams to create, manage, and share their development setups.

Usage

LSL provides a command-line interface (CLI) for managing lightweight Docker containers. Here's a breakdown of the available options:

  • lsl --list or lsl -l: Lists the available containers configured in containers.txt.
  • lsl -n : Starts a container with the specified name.
  • lsl --persist -n : Starts a container with data persistence enabled.
  • lsl --net -n : Starts a container using the host network.

Examples:

lsl -l
lsl -n alpine
lsl --persist -n web_app
lsl --net -n db_server

Configuration

LSL relies on a containers.txt file to define available containers and their corresponding Docker images.

containers.txt Format

# Example format: container_name = docker_repository:tag
# Uncomment or add entries below:
alpine = alpine:latest
ubuntu = ubuntu:22.04
# myapp = yourusername/yourimage:version

Each line should follow the format container_name = docker_repository:tag. Comments start with #.
